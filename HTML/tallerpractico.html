Punto 3: Coloreando enlaces

Pregunta: Si aplicamos color rojo a un <aside> con clase .links, ¿qué quedará en rojo?

Respuesta: Todo el texto dentro del aside, incluyendo los enlaces, porque el color se hereda. Sin embargo, si los enlaces tienen un color propio definido, ese estilo puede sobrescribir la herencia.

Punto 5: Enlaces y CSS

Pregunta 1: ¿Por qué los enlaces son verdes y no rojos?

Porque existe una regla CSS específica para los enlaces que tiene más prioridad que la herencia.

Pregunta 2: ¿Cómo hacer que los enlaces sean rojos?

Aplicando: a { color: red; }

Pregunta 3: ¿Qué pasa si cambiamos <ul> y <li> por <div>?

Se pierde la semántica de lista, aunque visualmente puede verse igual.

Pregunta 4: ¿Cómo rehacer el CSS sin tocar el original?

Usando clases nuevas para reemplazar los selectores sin modificar el código base.
